KEY METRICS AT A GLANCE
================================================================================

PLATFORM SCALE:
  • Documents/day: 10,000 (with 2x capacity = 20K)
  • Entities/day: 50,000 (with entity mapping challenges)
  • Concurrent users: 500+
  • Extraction workers: 5-20 (auto-scaled on queue lag)
  • Microservices: 6 (sourcing, extraction, rules, workflow, approval, disseminate)

PERFORMANCE TARGETS:
  • UI response time: <2s (P95: 1.5s realistic)
  • Document extraction: <5 mins (P95: 3 mins realistic)
  • Entity mapping latency: <1s (cached), 500ms typical
  • Approval action: <1s
  • Uptime: 99.5% (99.0% realistic with discipline)

TECHNOLOGY STACK:
  • Language: Java (Spring Boot microservices)
  • UI: Angular 14+
  • Databases: PostgreSQL, MongoDB, Redis, Elasticsearch
  • Queue: Apache Kafka (1000 events/sec)
  • Batch: Apache Spark (data lake, ETL)
  • Orchestration: Kubernetes + Istio (service mesh)
  • Workflow: Apache Camunda (BPMN)
  • Observability: Jaeger (tracing), Prometheus (metrics), Splunk (logs)

ARCHITECTURE PATTERNS USED:
  • Event Sourcing (immutable event log)
  • CQRS (separate read/write models)
  • Saga (distributed transactions)
  • Circuit Breaker (fault tolerance)
  • Cache-Aside (caching pattern)
  • Bulkhead (thread isolation)
  • Retry with Exponential Backoff
  • Dead Letter Queue (failed message handling)
  • Blue-Green Deployment (zero-downtime)
  • Service Mesh (Istio for mTLS, tracing)

COST ESTIMATE:
  • Monthly: ~$2,480
  • Annual: ~$29,760
  • Per-document: ~$0.25 (at 10K docs/day)
